### Task 5: Desktop nav — in-flow tabs + blue underline + ghost hover

Must be inserted BEFORE the current "Task 4: Final verification" (renumber Task 4 → Task 5).

**What this task implements:**

#### Problem
Current desktop nav: `position: absolute; inset: 0 0 auto auto` inside `.main-content` — the navbar floats over the content area. Articles use `position: absolute; inset: 0` to fill `.main-content`, which means the nav and content overlap. The nav also uses a yellow `var(--orange-yellow-crayola)` 1px underline — wrong accent color (blue is the interactive signal).

#### Solution

**1. Wrap articles in `.articles-wrap` in `src/App.tsx`**

```tsx
// BEFORE (in App.tsx):
<div className="main-content" ref={contentRef}>
  <NavigationTabs activePage={lastActivePage.current} onPageChange={handleNavigate} />
  <About isActive={activePage === 'about'} isExiting={exitingPage === 'about'} />
  <Resume isActive={activePage === 'resume'} isExiting={exitingPage === 'resume'} />
  <Portfolio isActive={activePage === 'portfolio'} isExiting={exitingPage === 'portfolio'} />
</div>

// AFTER:
<div className="main-content" ref={contentRef}>
  <NavigationTabs activePage={lastActivePage.current} onPageChange={handleNavigate} />
  <div className="articles-wrap">
    <About isActive={activePage === 'about'} isExiting={exitingPage === 'about'} />
    <Resume isActive={activePage === 'resume'} isExiting={exitingPage === 'resume'} />
    <Portfolio isActive={activePage === 'portfolio'} isExiting={exitingPage === 'portfolio'} />
  </div>
</div>
```

**2. CSS changes in `src/App.css`**

`.main-content` — add flex column so nav stacks above articles-wrap:
```css
.main-content {
  position: relative;
  overflow: hidden;
  display: flex;
  flex-direction: column;
}
```

`.articles-wrap` (new rule, base styles):
```css
.articles-wrap {
  position: relative;
  flex: 1;
  overflow: hidden;
  min-height: 0;
}
```

Articles already use `position: absolute; inset: 0` (hidden state) and `position: relative` (active state) — these now apply inside `.articles-wrap` instead of `.main-content`. No changes needed to article rules.

Desktop navbar override at `@media (min-width: 1250px)` — replace the existing `.navbar` block:
```css
  .navbar {
    position: relative;      /* was: absolute; inset: 0 0 auto auto */
    width: 100%;
    display: flex;
    justify-content: flex-end;
    border-radius: 0;
    border-top: none;
    border-bottom: 1px solid var(--jet);
    background: transparent;
    padding: 0;
    box-shadow: none;
    flex-shrink: 0;
  }
```

Blue active underline — update the `@media (min-width: 1250px)` `.navbar-link::after` rule:
```css
  .navbar-link::after {
    content: "";
    position: absolute;
    bottom: 0;
    left: 0;          /* was: left: 18px — now full tab width */
    right: 0;         /* was: right: 18px */
    height: 2px;      /* was: 1px */
    background: oklch(65% 0.18 220);   /* was: var(--orange-yellow-crayola) */
    transform: scaleX(0);
    transform-origin: left;
    transition: transform 0.28s cubic-bezier(0.16, 1, 0.3, 1);
  }
```

Ghost underline on hover — add new rule in `@media (min-width: 1250px)`:
```css
  .navbar-link::before {
    content: "";
    position: absolute;
    bottom: 0;
    left: 0;
    right: 0;
    height: 2px;
    background: oklch(65% 0.18 220 / 0.28);
    transform: scaleX(0);
    transform-origin: left;
    transition: transform 0.22s cubic-bezier(0.16, 1, 0.3, 1);
    border-radius: 1px 1px 0 0;
  }
  .navbar-link:not(.active):hover::before {
    transform: scaleX(1);
  }
```

**3. Sync timing — already free, no JS changes needed**
`lastActivePage.current` stays on the old tab while `activePage === null` (during exit + blank pause). The `.active` class switches exactly when `activePage` becomes the new page — same moment content starts entering. Option B timing is already implemented.

**Verification for Task 5:**
- Desktop (≥ 1250px): nav sits in normal flow above the article area — no overlap
- Active tab: white text + 2px blue underline spanning full tab width, scaleX animates in
- Hover inactive tab: ghost underline (30% opacity) slides in, disappears on mouseout
- Tab switch: underline stays on old tab during exit, moves to new tab when new content starts fading in (already works via React state)
- Mobile/tablet (< 1250px): no regression — navbar is still sticky-top as designed in Tasks 1–2

<decisions_made>
**Mobile layout:**
- Sidebar collapses to compact static header (scrolls away) at < 1250px
- Nav tabs sticky at top on mobile/tablet, replacing fixed bottom navbar
- Contact info (email, location, LinkedIn, GitHub) moves to bottom of About tab on mobile — hidden on desktop
- Static (not sticky) compact header preferred for more content space

**Desktop nav:**
- Blue 2px underline (not yellow) — yellow is for labels/categories, blue is for interactive/active state
- Ghost underline on hover (Option B) — 30% opacity, same scaleX mechanic, previews selection
- Underline timing Option B — syncs with content enter (already free via lastActivePage logic)
- Tabs in normal document flow ABOVE content (not overlapping) — requires `.articles-wrap` wrapper
</decisions_made>
